    NVM이란?

    • NVM(Node Version Manager)은 하나의 컴퓨터 환경에 여러 버전의 Node.js를 설치하고, 프로젝트에 맞춰 손쉽게 전환(Switching)하여 사용할 수 있게 해주는 도구이다.
    • 프로젝트마다 필요한 Node.js 버전이 다를 때 필수적인 버전 관리 프로그램으로, 명령어를 통해 간단히 설치하고 삭제 및 버전을 변경할 수 있다.

    설치

    • Window 버전은 Corey Butler nvm-windows Github 에서 다운로드 받아 사용할 수 있다.
    • 설치는 레지스트리를 건들지 않는 Portable 설치를 선호하기 때문에, “nvm-noinstall.zip”으로 설치한다.

    setting.txt

    root: D:\Development\Bin\Nvm
path: D:\Development\Bin\NodeJS
arch: 64
proxy: none
    속성 의미
    root NVM이 설치된 폴더의 경로를 의미하며, 해당 “setting.txt” 또한 동일 폴더 내 위치해야한다.
    path NVM으로 선택된 버전의 Node.js 폴더 심볼릭 링크 폴더로, 빈 폴더의 절대 경로로 설정한다.
    arch NVM으로 설치할 Node.js의 아키텍처를 설정한다.
    proxy NVM으로 Node.js를 설치할 때 사용할 프록시를 설정한다.

    시스템 환경 변수

    NVM_HOME=D:\Development\Bin\Nvm
NVM_SYMLINK=D:\Development\Bin\NodeJS
PATH=%NVM_HOME%;%NVM_SYMLINK%
    • “설정 > 시스템 환경 변수”에서 NVM_HOME, NVM_SYMLINK 설정을 추가한다.
    • 추가한 설정을 PATH 설정 내 %NVM_HOME%, %NVM_SYMLINK% 항목을 추가한다.

    Node 설치

    온라인 설치

    nvm install 20
nvm use 20
    • 설치할 버전을 명시해서 다운로드를 실행하면 LTS 버전으로 다운로드 받아진다.

    오프라인 설치

    • 오프라인 환경의 경우, 온라인 환경에서 Node.js 다운로드에서 환경에 맞는 Node.js 버전을 Standalone Binary(.zip)으로 다운로드 받는다.
    • 오프라인 환경의 NVM_HOME으로 설정된 경로에 노드 버전으로 넣어준다. (node-v20.20.0-win-x64.zip -> v20.20.0) 
      nvm use 20

    공통

    • 동일 버전을 use 명령어를 통해서 대표 Node.js 버전으로 설정하여 사용한다.
    • 필요 시, 다른 버전을 동일하게 설정 후 use 명령어로 환경 설정을 변경하며 작업하면 된다.
